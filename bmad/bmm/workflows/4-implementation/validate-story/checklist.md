# Validate Story - Validation Checklist

## Structure Validation

- [ ] Story file exists and is readable
- [ ] Story header section present
- [ ] Story statement (As a... I want... So that...) present
- [ ] Problem Definition section exists
- [ ] Solution Results section exists
- [ ] Metrics section exists
- [ ] Acceptance Criteria section exists

## Single Requirement Principle

- [ ] Problem Definition contains exactly ONE requirement
- [ ] Requirement is specific and focused
- [ ] No multiple requirements combined
- [ ] Requirement is clear and unambiguous

## Problem-Solution-Metrics Lifecycle

- [ ] Problem Definition is clear and specific
- [ ] Problem Definition is measurable
- [ ] Solution Results align with Problem Definition (if implemented)
- [ ] Metrics are defined and relevant
- [ ] Metrics are measurable
- [ ] Lifecycle is traceable (Problem ? Solution ? Metrics)

## Acceptance Criteria

- [ ] Acceptance criteria exist
- [ ] Criteria are specific and testable
- [ ] Criteria map to problem definition
- [ ] Criteria are complete

## Vibe Kanban Integration (Optional but Recommended)

- [ ] Ticket ID reference exists (recommended)
- [ ] Ticket exists in Vibe Kanban (if ticket_id present)
- [ ] Ticket status aligns with story status (if ticket_id present)
- [ ] 1:1 mapping maintained (if ticket_id present)
