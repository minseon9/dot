# Define Requirement - Validation Checklist

## Requirement Definition

- [ ] Problem description is clear
- [ ] Requirement contains exactly ONE problem
- [ ] Problem is specific and measurable
- [ ] Success criteria are defined

## Problem Statement

- [ ] Problem statement identifies who has the problem
- [ ] Problem statement describes what the problem is
- [ ] Problem statement explains why it's a problem
- [ ] Problem is solvable

## Metrics Definition

- [ ] Metrics are relevant to the problem
- [ ] Metrics are measurable
- [ ] Baseline is defined (or "to be measured")
- [ ] Target is defined (or "to be determined")
- [ ] Measurement method is identified

## Story Structure

- [ ] Story file created
- [ ] Story key is unique
- [ ] Epic assignment is correct
- [ ] Story number is assigned
- [ ] Problem Definition section populated
- [ ] Solution Results section exists (empty initially)
- [ ] Metrics section populated
- [ ] Acceptance Criteria section populated

## Integration

- [ ] Story added to sprint-status.yaml
- [ ] Story status set to "backlog"
- [ ] Epic context is correct
