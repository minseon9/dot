# Create Design Policy - Validation Checklist

## Story Analysis

- [ ] Story file loaded and understood
- [ ] Requirements identified
- [ ] Design-relevant requirements extracted

## Design System Understanding

- [ ] Design system loaded (Figma or doc)
- [ ] Design system patterns understood
- [ ] Constraints identified

## Policy Creation

- [ ] Policies created for all requirements
- [ ] Policies are specific and actionable
- [ ] Policies are traceable to requirements
- [ ] Policies organized by category

## Design System Alignment

- [ ] Policies use design system components
- [ ] Policies follow design system patterns
- [ ] Conflicts identified and resolved
- [ ] Extensions documented if needed

## Story Integration

- [ ] Policy file saved
- [ ] Story updated with policy reference
- [ ] Link to policy file added
